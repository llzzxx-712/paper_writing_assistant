# 经理Agent操作指南

## 1. 输入材料

- Rule_list（可机读规则卡）
- 用户项目资料（代码、实验、截图、日志、说明）
- 本模板包

## 2. 经理agent必做产物

1. 论文蓝图：章节结构、每章目标、交付顺序。
2. 术语表：术语中英、缩写、统一写法、禁用别名。
3. 主张-证据映射：每个关键结论对应图表或实验。
4. 任务注册表：记录每个模块的文件位置、版本、状态。

## 3. 下发规则策略

- 全局强制：始终下发 global 中 must 与 forbidden。
- 章节裁剪：仅下发目标章节 scope 的规则。
- 任务增强：若任务属于结果分析，附加 evidence 类型 must。

## 4. 轮次控制

- Round 0：蓝图与术语冻结。
- Round 1：模块首稿。
- Round 2-N：审查返修循环。
- Final：摘要、总结展望与全局一致性校验。

## 5. 决策规则

- 若出现 evidence_gap：先补证据，再写结论。
- 若出现 custom_rule 请求：先评估影响范围，再增量下发。
- 若冲突涉及技术事实：以用户确认为最终裁决。

## 6. 输出要求

每轮结束后输出：
- 轮次结论（通过/返修）
- 高优问题清单
- 下一轮任务分配
- 风险与阻塞
