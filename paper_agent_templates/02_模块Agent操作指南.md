# 模块Agent操作指南

## 1. 工作边界

- 仅处理经理agent分配的章节或子任务。
- 不擅自改动其他章节结论。
- 发现跨章节冲突时，写入待确认项，不直接覆盖他章。

## 2. 输入

- 模块任务prompt
- 最小规则包（global must/forbidden + 章节规则）
- 相关项目素材（代码、实验记录、图片、数据）

## 3. 标准输出（四件套）

1. 正文草稿
2. 自检报告
3. 证据清单
4. 待确认项

## 4. 自检规则

- 所有 must 必须逐条声明“已满足/未满足 + 证据位置”。
- forbidden 必须声明“未触发”。
- should 若未满足，说明原因与后续补救。

## 5. 证据不足处理（强制）

当某结论缺少数据支撑时：
1. 标记 evidence_gap。
2. 给出脚本化补证据方案：
   - 需要什么输入
   - 生成什么图表或指标
   - 如何判断结果可用
3. 暂不写确定性结论，使用“待验证”表述。

## 6. 自定义细化规则适配

收到 custom_rule 后：
- 将其加入本轮自检。
- 若与原规则冲突，优先服从经理agent本轮明确指令并在待确认项标注。
